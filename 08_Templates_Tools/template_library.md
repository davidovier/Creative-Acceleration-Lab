# Template Library

## Documentation Templates

### 1. Prototype Report Template

**Structure:**
```
# [Project Name] Prototype Report

## Executive Summary
- Project overview
- Core problem addressed
- Key stakeholders

## Emotional Core
- Founder's emotional driver
- User pain points
- Transformation goals

## Story Framework
- Hero's journey mapping
- Narrative arc
- Product metaphor

## Technical Overview
- Architecture decisions
- Data models
- Technology stack
- AI integrations

## UX Design
- User flows
- Key screens/wireframes
- Interaction patterns

## Prototype Deliverables
- What was built
- What was simulated/mocked
- Technical constraints

## Success Metrics
- How we measure impact
- Key performance indicators
- User feedback goals

## Next Steps
- Immediate actions
- Phase 2 recommendations
- Long-term roadmap
```

### 6. Audit Report Template

**Structure:**
```
# [Company] Acceleration Audit Report

## Executive Summary
- Current state overview
- Key findings
- Critical recommendations

## Bottleneck Map
- Where work gets stuck
- Process inefficiencies
- Decision delays

## Bureaucracy Index
- Unnecessary approvals
- Redundant processes
- Complexity score

## Team Emotional State
- Morale assessment
- Frustration points
- Energy levels

## Recommendations
### Quick Wins
- Immediate changes (< 1 week)
- Low effort, high impact

### Strategic Changes
- Medium-term improvements (1-3 months)
- Process redesigns

### Cultural Shifts
- Long-term transformations
- Mindset and value changes

## Implementation Roadmap
- Prioritized action items
- Resource requirements
- Success metrics
```

### 7. Client Onboarding Form

**Fields:**
```
# Client Onboarding Form

## Basic Information
- Name
- Company/Project
- Contact details

## Project Context
- What are you building/creating?
- Who is it for?
- What problem does it solve?

## Emotional Drivers
- What brought you to this project?
- What keeps you up at night about it?
- What would success feel like?

## Core Wound/Struggle
- What past experience drives this?
- What are you trying to prove or heal?

## Desired Transformation
- What do you want to become through this?
- How should this change you or your users?

## Symbol Preference
- Are there visual symbols that resonate with you?
- Colors, shapes, imagery that feel right?

## Technical Context
- Current tech stack (if applicable)
- Technical constraints
- Team capabilities

## Timeline & Constraints
- Ideal timeline
- Budget considerations
- Must-haves vs. nice-to-haves

## Archetype Self-Assessment
- Which archetype(s) do you identify with?
- How do you see your brand personality?
```
